# Шаг 4 — HDMI с переключением паттернов кнопками

Languages: [English](README.md) · **Русский**

Шаг 3 вывел статичную картинку. Теперь добавляется интерактивность: четыре кнопки
шилда меняют то, что рисуется на экране. Видеопуть тот же, что в шаге 3 (PS7 →
FCLK0 → MMCM → rgb2dvi), рецепт прошивки тот же — только генератор паттернов стал богаче.

## Что на экране

Тёмно-синее **окно 960×720 по центру экрана** с прыгающим квадратом 144×144 (режим
по умолчанию), цвет которого меняется каждый кадр. Четыре кнопки:

| Кнопка | Пин | Действие |
|--------|-----|----------|
| KEY1   | P19 | сменить паттерн: квадрат → цветные полосы → градиент → шахматная клетка |
| KEY2   | T19 | инвертировать цвета |
| KEY3   | U20 | замедлить анимацию |
| KEY4   | U19 | ускорить анимацию |

Кнопки срабатывают по фронту с антидребезгом ~14 мс, так что одно нажатие — одно
действие (не «удерживать для продолжения», как в шаге 2). Назначение — по пину FPGA;
физические метки KEY на шилде могут отличаться (см. [Шаг 0](../00-setup/#expansion-board-reference)).

## Почему окно по центру, а не на весь экран

HDMI-сигнал полноформатный, 1280×720, но демо рисует контент только внутри
центрального окна **960×720** (ровно **4:3**), а боковые полосы заливает чёрным —
«pillarboxing». Ретро-контент — 4:3; растянуть его на 16:9-панель значит сделать
всё широким и приплюснутым, поэтому чёрные боковины сохраняют пропорции, как на CRT.
Чтобы использовать всю ширину, достаточно выставить `WX0=0` и `WW=1280` в
[`pattern_gen3.v`](pattern_gen3.v) и пересобрать.

## Build

Та же блочная схема, что в шаге 3 (переиспользует `clkgen2.v`, `clk_wiz` MMCM и
`rgb2dvi` от Digilent), с [`pattern_gen3.v`](pattern_gen3.v) в качестве источника
паттернов и [`hdmi_btn720.xdc`](hdmi_btn720.xdc), добавляющим четыре пина кнопок.

```
export VIVADO_LIBRARY=~/vivado-library
vivado -mode batch -source build_720pl_z010.tcl
```

Часть `xc7z010clg400-1`, размер выхода ~2 083 867 байт. Готовый `hdmi720_z010.bit`
прилагается.

## Flash

Тактирование от PS, как в шаге 3, поэтому используется тот же надёжный двухэтапный
рецепт (программировать PL через `vivado_lab`, затем `ps7_init` + `ps7_post_config`
через `xsdb`):

```
bash flash_demo.sh hdmi720_z010.bit
```

`End of startup status: HIGH`, затем `PS7_INIT_DONE` и `PS7_POST_CONFIG_DONE`.
Причины такой последовательности — в [Шаге 3](../03-hdmi-bars/), настройка JTAG —
в [Шаге 0](../00-setup/). При штатной загрузке с SD FSBL сам поднимает тактирование PS.

## Ожидаемый результат

Прыгающий квадрат появляется в центральном окне, H18 мигает. KEY1 переключает между
четырьмя паттернами, KEY2 инвертирует, KEY3/KEY4 регулируют скорость. Ввод плюс
настоящая картинка — плата теперь по-настоящему интерактивна.
